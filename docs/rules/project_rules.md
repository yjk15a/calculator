# Project Rules

## Core Logic Development
- **TDD (Test-Driven Development)**: All core logic must be developed using TDD.
  - Write a failing test first.
  - Write the minimum code to pass the test.
  - Refactor while keeping tests passing.
- **SOLID Principles**: Adhere to SOLID principles for all core logic.
  - Single Responsibility Principle
  - Open/Closed Principle
  - Liskov Substitution Principle
  - Interface Segregation Principle
  - Dependency Inversion Principle

## UI Development
- UI code is excluded from strict TDD requirements but must be clean and maintainable.
